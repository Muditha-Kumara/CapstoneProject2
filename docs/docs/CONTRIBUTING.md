Thank you for considering contributing to this project.
These guidelines explain how to propose changes, create branches, write commits, and submit pull requests.

1. Getting Started

Fork the repository to your GitHub account

Clone the fork to your local machine

Create a feature branch before making changes

Example:

git checkout -b issue29-docs

2. Branch Naming Rules

Use clear and descriptive names:

issuexx-description

feature-frontend-update

bugfix-login-form

3. Commit Guidelines

Write meaningful commit messages.

Examples:

Add user manual documentation (Issue #29)
Fix button alignment on dashboard
Refactor request form component


A good commit message includes:

What you changed

Why you changed it

The related issue number

4. Pull Request Process
Steps:

Ensure your branch is up to date with main

Push your branch to your fork

Open a pull request to the main repository

Add a clear title and description

Request a review if needed

5. Code Quality

Follow formatting rules from frontend_guidelines.md

Keep code readable and modular

Use comments for complex logic

Avoid unnecessary files in commits

6. Testing (If applicable)

Test your changes before submitting a PR.

7. Where to Ask Questions

Use GitHub Issues or project communication channels.

8. Summary

These guidelines help keep contributions organized, readable, and easy to review.
